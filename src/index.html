Installation

Installation Steps

Prerequisites:

  • Ensure you have Node.js and npm installed on your system. You can download and install them from nodejs.org.
  • Verify the installation by running the following commands in your terminal or command prompt: 
                     
  node -v 
  npm -v

Navigate to Project Directory:

  • If you downloaded the ZIP file, navigate to the extracted folder in your terminal or command prompt.
  • After navigate to the extracted folder, you have to navigate to the template folder

Install Dependency:

  • Run the following command to install the required dependencies using npm: 
                     
  npm install

Run the Development Server:

  • Once the installation of dependencies is complete, start the development server by running: 
                     
  npm run dev
  • This command will build and serve your Next.js application locally.
  • Open your web browser and navigate to http://localhost:3000 to view your website.

Following these steps should get your Next.js website template up and running locally, allowing you to start developing and customizing it according to your requirements.

Project Structure

Folder Structure

The template package you receive will contain 2 directories.

  • documentation
  • template

Documentation:

From Here you can learn everything about this template and understand which file is responsible for what and how to customize this template.

Template:

This is the main template which you will be use. Here is how i organize my folder structure for this template.

  • src : This folder contains all the projects code.
  • public : The files placed in the "public" folder are served as static assets by Next.js. These files are accessible directly by their URLs relative to the root of the website. For example, if you have an image named "logo.png" in the "public" folder, it would be accessible at /logo.png.
  • next.config.js : Configuration file for Next.js
  • package.json : Project dependencies and scripts
  • .eslintrc.json : Configuration file for ESLint
  • .gitignore : Git files and folders to ignore
  • next-env.d.ts : TypeScript declaration file for Next.js
  • tsconfig.json : Configuration file for TypeScript
  • tsconfig.json : Configuration file for TypeScript
  • tailwind.config.ts : Configuration file for Tailwind Css
  • postcss.config.js : Configuration file for Postcss
  • component.json : Configuration file for Shadcn UI

Inside the src directory we have organize our folder structure like this.

  • app : This directory contains all the routing in nextjs.
  • components : This will contain all the components of this template

Header

How to update navbar?

To update navigation bar of this template you have to navigate to src > app > components > navbar.tsx . From here you can customize the navbar.

How to update navbar logo?

To update navigation bar logo, you have to navigate to public folder. Here you have to put your logo file which name have to be logo . If your logo extension is .svg, then you don't have to update any code on the navbar.tsx file. But if your extension is not .svg, then you have to modify file extension on here.

If you want to update the chapters content, you have to navigate to src > lib > data > navigation-data.ts 

                
  export const chapters = [
    {
      id: 1,
      label: "Chapters",
      href: "#chapters",
    },
  ]

Here you can add you own navigation by using this format.

Note: If you update any href of any existing navigation links, then you have to update the id of the specific sections which href you updated.
